# OSU MPI Benchmark Suite

Automated benchmark runner, collector, and plotter for comparing MPI
installations (UCX, HCOLL, UCC, IntelMPI, etc.) using the OSU
Micro-Benchmarks. Supports Slurm and direct execution.

## Directory structure

```
osu-bench-suite/
├── config.sh                  ← Edit this first
├── scripts/
│   ├── run_collectives.sh     ← Run collective benchmarks
│   ├── run_pt2pt.sh           ← Run point-to-point benchmarks
│   ├── collect_results.sh     ← Parse results into CSV
│   ├── plot_results.py        ← Generate plots
│   └── check_jobs.sh          ← Check Slurm job status
├── results/
│   ├── <mpi_label>/
│   │   └── <benchmark>/
│   │       ├── nodes1.dat
│   │       ├── nodes2.dat
│   │       └── pt2pt.dat
│   ├── csv/                   ← Generated by collect_results.sh
│   └── logs_<timestamp>/      ← Slurm stdout/stderr
└── plots/                     ← Generated by plot_results.py
```

## Setup

### 1. Edit `config.sh`

```bash
# Define MPI installations to compare
MPI_INSTALLATIONS=(
    ["openmpi-ucx"]="/path/to/openmpi-ucx/bin/mpirun"
    ["openmpi-hcoll"]="/path/to/openmpi-hcoll/bin/mpirun"
    ["openmpi-ucc"]="/path/to/openmpi-ucc/bin/mpirun"
)

# Set extra mpirun flags per installation
MPI_EXTRA_FLAGS=(
    ["openmpi-hcoll"]="--mca coll_hcoll_enable 1 --mca pml ucx"
    ["openmpi-ucc"]="--mca coll_ucc_enable 1 --mca pml ucx"
)

# Match your node's core count
RANKS_PER_NODE=128

# Hostfile with one hostname per line (8+ nodes for full sweep)
HOSTFILE=./hostfile

# Slurm partition
SLURM_PARTITION=compute
```

### 2. Build OSU Micro-Benchmarks

The build script downloads OMB and builds it **once per MPI installation**,
so each set of binaries is linked against the correct MPI library.

```bash
chmod +x scripts/*.sh

# Build for all MPI installations in config.sh (downloads OMB 7.5.1)
./scripts/build_osu.sh

# Options:
./scripts/build_osu.sh --version 7.5.1        # specific version
./scripts/build_osu.sh --mpi openmpi-hcoll    # one installation only
./scripts/build_osu.sh --with-cuda            # enable GPU buffer benchmarks
./scripts/build_osu.sh --cuda-home /usr/local/cuda
./scripts/build_osu.sh --jobs 16              # parallel make jobs
./scripts/build_osu.sh --dry-run              # preview commands
```

After a successful build, `config.sh` is automatically updated with an
`OSU_INSTALLATIONS` array mapping each MPI label to its install path.
The run scripts use this automatically — no manual path editing needed.

Build outputs are placed under `./osu-builds/<label>/`:
```
osu-builds/
├── src/
│   └── osu-micro-benchmarks-7.5.1.tar.gz    ← downloaded once
├── openmpi-ucx/
│   ├── configure.log
│   ├── make.log
│   └── osu-micro-benchmarks-7.5.1/
│       └── libexec/osu-micro-benchmarks/mpi/
├── openmpi-hcoll/
│   └── ...
└── openmpi-ucc/
    └── ...
```

## Usage

### Full workflow

```bash
chmod +x scripts/*.sh

# 1. Build OSU for each MPI installation (downloads + compiles)
./scripts/build_osu.sh

# 2. Run benchmarks
./scripts/run_collectives.sh
./scripts/run_pt2pt.sh
```

### Run a single MPI installation or benchmark

```bash
# Only HCOLL, only allreduce, only 4 nodes
./scripts/run_collectives.sh --mpi openmpi-hcoll --bench osu_allreduce --nodes 4

# Dry-run to preview commands
./scripts/run_collectives.sh --dry-run
```

### Check Slurm job status

```bash
./scripts/check_jobs.sh
```

### Collect and plot results

```bash
# After all jobs complete:
./scripts/collect_results.sh

# Generate all plots (PNG by default)
python3 scripts/plot_results.py

# PDF output, specific benchmark only
python3 scripts/plot_results.py --format pdf --bench osu_allreduce
```

## Plots generated

For each benchmark:

| Plot | Filename | Description |
|------|----------|-------------|
| vs Message Size | `<bench>_vs_msgsize.png` | Latency/BW curves, one panel per node count |
| Scaling | `<bench>_scaling.png` | Latency/BW at fixed message sizes vs node count |
| Heatmap | `<bench>_heatmap.png` | MPI label × node count at largest message size |
| Overview | `comparison_overview.png` | Bar chart comparing all MPI installs across all benchmarks |

## Collectives tested

| Benchmark | What it measures |
|-----------|-----------------|
| osu_allreduce | All-reduce latency — key for ML and iterative solvers |
| osu_alltoall | All-to-all latency — critical for FFT and transpose |
| osu_allgather | All-gather — used in halo exchanges and data replication |
| osu_bcast | Broadcast from root to all ranks |
| osu_barrier | Global synchronization overhead |
| osu_reduce | Reduction to single root |
| osu_reduce_scatter | Reduce + scatter in one step |
| osu_alltoallv | Variable-length all-to-all |

## Point-to-point tested

| Benchmark | What it measures |
|-----------|-----------------|
| osu_latency | Ping-pong latency between 2 ranks |
| osu_bw | Unidirectional bandwidth |
| osu_bibw | Bidirectional bandwidth |
| osu_mbw_mr | Multi-pair bandwidth × message rate |
| osu_latency_mp | Multi-pair latency |

## MCA flags reference for HCOLL / UCC

```bash
# Force HCOLL for all collectives
--mca coll_hcoll_enable 1 --mca pml ucx

# Force UCC
--mca coll_ucc_enable 1 --mca pml ucx

# Disable both (baseline UCX only)
--mca coll_hcoll_enable 0 --mca coll_ucc_enable 0 --mca pml ucx

# Verbose collective selection (debug which plugin is chosen)
--mca coll_base_verbose 10
```

## Tips

- Run `--dry-run` first to validate commands before submitting jobs.
- For HCOLL ABI issues, ensure the HCOLL library matches the OpenMPI build.
- Use `OMPI_MCA_*` environment variables as an alternative to `--mca` flags.
- Add `--mca coll_base_verbose 10` to confirm which collective module wins.
